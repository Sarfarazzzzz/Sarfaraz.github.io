## 🚀 Featured Builds

### ☁️ [AWS CloudGuide - Conversational AI](https://github.com/Sarfarazzzzz/Final-Project-Group-BotBuilders/tree/main)
**The Problem:** Navigating 100k+ pages of AWS documentation to find specific architectural answers is a nightmare. 
**The Build:** I architected a hybrid RAG system to solve semantic ambiguity across massive documentation sets. Fusing dense (bge-m3) and sparse (BM25) retrieval, and backing it with a batched PyTorch GPU pipeline handling 800k+ vectors, I deployed a conversational AI (Qwen-2.5-7B) that delivers cited, highly accurate answers in under 200ms.
> **Tech:** Python, LangChain, AWS, PyTorch, Hugging Face

### 🎨 [Auto Image Restoration & Colorization](https://final-project-group3demo.streamlit.app/)
**The Problem:** Restoring degraded, black-and-white images requires overcoming severe regression-to-the-mean biases.
**The Build:** I orchestrated a scalable deep learning pipeline on AWS EC2 processing 118k images. By pioneering a ResNet-18 U-Net and VAE-regularized framework, I achieved 85% model compression without sacrificing fidelity (0.496 SSIM). I then launched a real-time Streamlit app featuring a custom saturation boosting technique to bring the images to life.
> **Tech:** PyTorch, AWS EC2, Streamlit, OpenCV

### 🏏 [IPL Live Match Win Predictor](https://iplprediction-tywz8ytnoa5carjvxla3w6.streamlit.app/)
**The Problem:** T20 cricket is notoriously volatile, making early-game win predictions practically useless.
**The Build:** I engineered a live, ball-by-ball win probability engine using an XGBoost model trained on over 500 historical matches. To tame the chaotic early-game data, I designed a custom confidence-weighted blend-in algorithm that successfully slashed forecast uncertainty by 60%.
> **Tech:** XGBoost, Python, Streamlit, Pandas
